# Передача контекста — Admin App / Telegram Bots / Сценарии

## Проект

Проект: Admin App  
Домен: `https://app.exec-booster.kz`  
Активный модуль: `Telegram Bots` / «Чат-боты»  
Активный раздел разработки: `Сценарии`

URL:

```text
/admin.php?tab=telegram_bots&page=scenarios
```

## Главные правила работы

```text
1. Работать микропатчами.
2. В ZIP класть только изменённые файлы.
3. В каждом ответе писать текущий шаг, что входит в патч, что проверять, что дальше.
4. Не возвращать старую тяжёлую страницу подписчиков.
5. Не делать тяжёлые COUNT/MAX/JOIN по сообщениям в списках.
6. Не делать runtime ALTER TABLE при открытии страниц.
7. Не ломать текущую мобильную верстку.
8. Не использовать синие кнопки.
9. Основной акцент — тёплый оранжевый, вторичные кнопки — серые/светлые.
10. Основную страницу списка сценариев больше не перестраивать без явного запроса.
```

## Что было ошибкой и больше не повторять

Был неудачный заход: canvas-редактор начали встраивать в основную страницу `pages/scenarios.php`. Это сломало утверждённый формат страницы.

Правильное разделение:

```text
pages/scenarios.php
  -> только список сценариев, фильтры, модальное создание/редактирование, архив

pages/scenario_editor.php
  -> будущий редактор блоков/canvas
```

## Следующий шаг

Следующий технический шаг:

```text
Шаг 3 из 10 — scenario_editor.php, редактор блоков отдельным файлом.
```

Что делать:

```text
1. Не трогать основной список сценариев в `pages/scenarios.php`.
2. Создать/доработать отдельный `pages/scenario_editor.php`.
3. Сделать вход в редактор из строки сценария через меню ⋯ или отдельный пункт позже.
4. На первом шаге редактора: поле, добавление блока «Сообщение», сохранение координат.
5. Не запускать сценарии и не отправлять сообщения до отдельного runner-патча.
```

## Последняя проблема, которую исправляли

В меню строки сценария кнопка ⋯ открывала плашку, но она обрезалась границей блока. Причина — `overflow:hidden` на контейнере страницы. Исправление — `overflow:visible` и повышенный `z-index` меню.
