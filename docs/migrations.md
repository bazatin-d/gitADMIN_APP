# SQL-миграции проекта

Миграции нужны, чтобы не держать SQL-запросы в переписке и не гадать, что уже было выполнено на сервере.

## Где лежат файлы

```text
database/migrations/   — структура БД
database/seeds/        — стартовые справочники и тестовые заполнения
```

## Реестр применённых миграций

```text
app_migrations
```

Первым выполняется файл:

```text
database/migrations/2026_05_27_000_create_app_migrations.sql
```

После этого каждая миграция добавляет себя в таблицу `app_migrations`.

## Как выполнять

1. Сделать экспорт БД.
2. Открыть phpMyAdmin.
3. Выполнить нужный файл из `database/migrations/`.
4. Проверить `/admin.php?tab=diagnostics`.

## Важное правило

Уже выполненный SQL-файл не редактируем. Если нужно добавить колонку, индекс или таблицу — создаём новый файл:

```text
YYYY_MM_DD_00N_what_changed.sql
```

Так у нас не будет ситуации: «я вроде это уже делал, но то ли в прошлом чате, то ли во сне».
