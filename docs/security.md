# Безопасность проекта

## Что уже включено

- Закрыт прямой доступ к `admin_app`, `b24_integration`, `docs`, `oca_interpretation`.
- Закрыт прямой доступ к архивам, SQL-дампам, логам, backup-файлам и dot-файлам.
- Для админки включены базовые HTTP-заголовки безопасности.
- Для сессии админки включены `HttpOnly`, `SameSite=Lax`, `Secure` при HTTPS и строгий режим сессий.
- Все POST-действия в авторизованной админке требуют CSRF-токен.
- CSRF-токен автоматически добавляется во все POST-формы админки при выводе HTML.

## Правила разработки

1. Новые POST-формы в админке можно делать обычным `<form method="POST">`; токен добавится автоматически.
2. Если форма отправляется через `fetch`/AJAX, токен нужно передавать вручную из скрытого поля `csrf_token` или из `asr_csrf_token()`.
3. Новые обработчики POST нужно размещать только в `admin_app/modules/{module}/actions.php`.
4. Публичные endpoint-файлы (`save.php`, `save_progress.php`, `public_settings.php`, `u.php`) не должны подключать админский `dispatcher.php`.
5. В публичных ответах нельзя показывать SQL-ошибки, webhook, пароли, пути сервера и stack trace.

## Что проверить после установки

- Вход в админку.
- Сохранение настроек.
- Создание/редактирование пользователя.
- Создание короткой ссылки.
- Редактирование UTM-значений.
- Ручной ввод результата.
- Импорт CSV.
- Отправка письма/обновление CRM.
