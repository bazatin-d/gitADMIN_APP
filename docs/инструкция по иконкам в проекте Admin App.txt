Инструкция по использованию иконок Admin App
1. Главный принцип

В Admin App используются только уже утверждённые SVG-иконки из папки:

assets/admin/icons/

Нельзя самовольно:

рисовать новые иконки в другом стиле;
использовать emoji как управляющие иконки интерфейса;
возвращать старые символы •, ✎, 🗑, 🔔, 👥, 🔒, ☰, ×, ▶, ▼;
использовать синие иконки;
использовать разноцветные иконки без явного решения.

Стиль иконок:

серый / графитовый;
толстая округлая линия;
без лишних деталей;
без синего цвета;
без яркого оранжевого внутри самих иконок, кроме случаев, где это уже принято в кнопке;
размер обычно 16–24 px;
в кнопках иконка и текст всегда в одну строку.
2. Как вставлять иконку

Основной способ - обычный <img>:

<img class="admin-icon" src="/assets/admin/icons/имя-иконки.svg" alt="" aria-hidden="true">

Для кнопки:

<button class="btn-with-icon">
    <img class="admin-icon admin-icon--sm" src="/assets/admin/icons/rs2-download-white.svg" alt="" aria-hidden="true">
    <span>Скачать</span>
</button>

Обязательное правило для кнопок с иконкой:

.btn-with-icon {
    display: inline-flex;
    flex-direction: row;
    align-items: center;
    justify-content: center;
    gap: 7px;
    white-space: nowrap;
    line-height: 1;
}

Иконка не должна становиться над текстом.

3. Что запрещено

Не использовать:

asr_admin_icon()

Не добавлять новые helper-функции для иконок.

Не подключать иконки через:

file_get_contents()

Не вставлять SVG inline через PHP.

Не подключать иконки глобально через:

bootstrap.php

Не менять bootstrap.php ради иконок.

Не делать массовую замену иконок по всему проекту одним патчем.

4. Размеры
Обычная иконка
.admin-icon {
    width: 18px;
    height: 18px;
    display: inline-block;
    vertical-align: middle;
    object-fit: contain;
    flex: 0 0 auto;
}
Маленькая иконка в кнопке
.admin-icon--sm {
    width: 14px;
    height: 14px;
}
Иконка заголовка блока
.admin-icon--title {
    width: 24px;
    height: 24px;
    flex: 0 0 24px;
}
Иконка в карточке / крупном блоке
.admin-icon--lg {
    width: 28px;
    height: 28px;
}
5. Иконка + заголовок

Заголовок с иконкой должен быть строго в одну строку:

<div class="section-title-with-icon">
    <img class="admin-icon admin-icon--title" src="/assets/admin/icons/tb2-telegram-gray.svg" alt="" aria-hidden="true">
    <div>
        <h3>Telegram-боты</h3>
        <p>Описание раздела</p>
    </div>
</div>

CSS:

.section-title-with-icon {
    display: flex;
    flex-direction: row;
    align-items: center;
    gap: 12px;
}

.section-title-with-icon h2,
.section-title-with-icon h3 {
    margin: 0;
    line-height: 1.15;
}

Если есть описание под заголовком, оно остаётся внутри правого блока, а не под иконкой.

15. Цвета
Серые иконки

Использовать для обычных действий:

-gray.svg
Белые иконки

Использовать только на тёмных или оранжевых кнопках:

-white.svg

Пример:

<button class="bg-[#FFA048] text-white">
    <img src="/assets/admin/icons/rs2-download-white.svg" alt="" aria-hidden="true">
    Скачать
</button>

Не ставить серую иконку на оранжевую кнопку, если она плохо читается.

16. Как выбирать иконку для нового модуля

При создании новой страницы или модуля:

Сначала определить смысл действия:
добавить;
сохранить;
удалить;
редактировать;
закрыть;
скачать;
загрузить;
ссылка;
пользователь;
настройки;
Telegram;
файл;
поиск.
Найти подходящую иконку в уже существующих наборах.
Если модуль похож на существующий, использовать его серию:
доступы - av2-*;
Telegram - tg2-* или tb2-*;
настройки - st2-*;
URL Shortener - sh2-*;
результаты / импорт - rs2-*;
меню - mn4-*;
модальные окна - mo2-*;
шапка - ly2-*.
Если подходящей иконки нет, не рисовать новую сразу. Сначала использовать ближайшую стандартную.
Новую иконку добавлять только отдельным маленьким патчем.
17. Правило патчей

Иконки внедрять только маленькими шагами.

Нельзя менять сразу:

bootstrap.php;
layout.php;
несколько модулей;
меню;
Telegram;
пользователей;
доступы.

Безопасный порядок:

1 модуль = 1 патч

После каждого патча проверять:

страница открывается;
нет белого экрана;
тёмная тема;
мобильная верстка;
кнопки не стали выше;
иконка и текст стоят в одну строку.
18. Чек-лист перед выдачей патча

Перед тем как дать патч, проверить:

php -l изменённые PHP-файлы

Проверить, что не добавлено:

file_get_contents
asr_admin_icon
новый helper
подключение в bootstrap.php
inline-SVG через PHP
emoji как иконка интерфейса

Проверить, что кнопки с иконками имеют:

display: inline-flex;
flex-direction: row;
align-items: center;
gap: 7px;
white-space: nowrap;
19. Что считать завершённым стандартом

Текущий стандарт иконок Admin App считается таким:

серые / графитовые SVG
толстая округлая линия
без синих элементов
без emoji в интерфейсе
через обычный img
маленькие патчи
без bootstrap.php
без asr_admin_icon()
без file_get_contents()

При создании новых модулей использовать только этот стандарт.
